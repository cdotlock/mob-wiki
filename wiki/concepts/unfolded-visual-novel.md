当前产品的展示形态是 Unfolded 风格互动视觉小说——一种面向竖屏移动端的沉浸式阅读体验。它不是"角色表演引擎"，而是"可视化阅读器"：用整屏背景图承载场景氛围，用放大角色 cutout 做情绪肖像，用悬浮式对话气泡承载文本，用点击推进节奏。

脚本格式规范见 [[concepts/mss-format]]。解释器见 [[entities/moonshort-script]]。

## 与传统 Episode 风格的根本差异

MobAI 不是 Episode 那种"强舞台调度感的站位系统"。核心差异：

- **场景是主画面，角色是叠加层**，而非角色站位编排的舞台
- **对话框悬浮在画面上方**，而非强占底部的大面积 UI 面板
- **同一时刻通常只强调一个角色**，而非长期维持多角色同屏
- **镜头语言依赖背景切换、淡入和角色滑入**，而非复杂的 blocking
- **内心独白、旁白、短信界面是同等级叙事容器**，和常规对白并列存在

体验更像一部会被你点击打断、偶尔需要做决定的互动漫画。

## 画布与分层

播放器使用固定上限宽度的竖屏容器：最大宽度 393px，高度 100dvh，居中，黑色背景。先服务竖屏移动端，再兼容桌面预览。

画面分层（从后到前）：

1. 背景图层（整屏 cover，带轻微平移制造镜头感）
2. 角色立绘层（底部裁切，强调上半身和情绪）
3. 对话区（悬浮式气泡）
4. 选项区（替代对话区出现）
5. 手机短信弹层
6. 点击提示

## 五种叙事容器

| 容器 | 视觉特征 | 用途 |
|------|---------|------|
| 常规对白 | 彩色 name tag + 白色气泡，左右分区 | 角色说话 |
| 内心独白 | thinking bubble 图标 + 白色气泡 | MC 心理活动（YOU） |
| 旁白 | 居中胶囊型文本块，无角色名 | 镜头切换、时间变化、情绪总结 |
| 手机短信 | iMessage 风格卡片弹层 | 秘密短信、夜间私聊、悬念推动 |
| 选择 | 纵向排列大按钮，替代对话框 | 分支决策点 |

容器切换本身就构成了阅读节奏——这是当前展示"好看"的来源，而非复杂动画。

## 交互节奏

### 点击推进

核心操作只有两种：点击屏幕继续阅读、在选项出现时点击按钮做选择。底部常驻 "Tap to continue" 脉冲文案。

### 自动推进 vs 手动推进

引擎把脚本步骤分两类：

**自动推进**（镜头动作，自己发生）：背景切换、角色显示/隐藏/换表情、程序等待、分支控制

**手动推进**（文本内容，必须玩家点击）：旁白、内心戏、对白、短信、选择

玩家感知到的是一个有镜头语言的阅读器，不是需要自己操心调度的系统。

### 角色出场节拍

新角色滑入画面后，系统停住等待玩家点一次。这让"角色出场"本身成为叙事事件——先看人，再看他说什么。

## 素材展示规则

### 背景图

使用 `background-image: cover` 铺满整屏。切换时"旧图保留 + 新图淡入 + 轻微横向平移"，制造镜头推移感。静态图即可，不需要视频背景。

可选时间变体：同一地点的早/中/晚/夜版本（色温滤镜）。

### 角色立绘

透明底 PNG，绝对定位在屏幕底部。`bottom: -45%`、`height: 130%`——角色被有意放大，下半身压到屏幕外，视觉焦点集中在上半身、脸部、姿态。更像情绪肖像，而非全身动作表演。

入场从画面边缘滑入（0.4s），退场滑出（0.3s）。表情切换直接换图，不做过渡。

### CG

关键剧情的全屏插画，替代常规背景+角色组合。3-5 秒 AI 短视频（Kling I2V），不做静态图+引擎动效。每部 60 集小说约 33-48 张 CG。

## 视觉资产管线

预生成（不是实时生成），这是和 DREAMIO/EndlessVN 等竞品的核心差异——Dramatizer 先出 60 集剧本，所有场景和角色已知，可以批量生成+人工 QA。

| 资产类型 | 方案 | 成本/部 |
|---------|------|---------|
| 角色 LoRA 训练 | 15-30 张参考图 | $8-40 |
| 角色静态立绘（含表情） | NovelAI/Flux + Inpainting | $2-4 |
| 角色 idle 动画 | LivePortrait → 精灵图序列 | 免费（开源） |
| 场景背景 | Gemini/Imagen | $1-6 |
| CG 短视频 | Kling 3.0 I2V | $21-50 |
| 峰值视频 | Kling/Runway | $4-37 |
| **合计** | | **$36-137** |

对比传统方案 $15K-84K，降低 100-600 倍。

## 游戏机制集成

D20 检定、小游戏、属性养成等游戏机制保留，嵌入阅读流而非独立阶段。通过 MSS 脚本的 `@minigame`、`@choice`、`@option brave/safe`、`check { attr / dc }` 指令集成。

详见 [[concepts/mss-format]] 中的游戏机制章节。
