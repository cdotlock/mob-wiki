# Moonshort 策略文档

## 大前提

1. 视频模型不及预期，长时间全自动化产线仍然不可行
2. 玩家体感和要求最重的是剧本效果
3. 如何解决营销和外部分发的难题

## 小前提

1. 成本控制
2. 小游戏可能打断心流
3. 潜在的 REMIX 体验割裂

## 我们的策略

1. 极其充沛的供应
2. 嵌入其中的二创体验

## 策略分析

目前市场主导者分为两类：

- **第一类**：Episode 代表的，精品度不够，角色定制程度更多，有大量二创和长期积累的粉丝，和完善的创作者生态，群体更幼，从 TikTok 上面引流更多
- **第二类**：Choices 代表的，PGC 为主的官方精品，质量上限更高，古法手作，也有很多忠实的粉丝群体，群体更成熟，从 Ins 上面引流更多

**我们的定位**：① 量更大 ② 定制化更多 ③ 二创体验 —— 在这些中，二创体验是杀手锏，也是 AI 带来的变量

我们在理论上能更优的地方就是充分利用玩家 UGC 的力量把 Episode 从被动的社区变成主动的社区，这让我们从单边提供内容的聚合平台，变为具有双边网络效应的平台。

## 新计划

### 1. 产品侧

#### A/B Test：复原 Live 2D + TTS 模式

复原 Live 2D + TTS 模式（后台已尽力保留此模式的兼容），并推出适宜这个模式的剧本，在原来的基础上做增强（建议去 demo 上体验文字版本）。

待定问题：
- 如果是原来的那种 live 2d（一张图的那种），要不要跑个简单的视频？
- 要不要做成 galgame 那种，场景图 + 前面立绘简单动效的形式？
- 要不要做成几张图片衔接剧情推进的形式？比如点到了某一句，就出第 N 张图

#### 内容上

- 剧可以十集十集地上，隔段时间更新一下，做成游戏卡带那样的感觉
- 针对 Live 2D，可能要有更适合的最终形态

#### 加速规划 REMIX 功能

- **剧情为主导的 REMIX 功能** —— 玩家二创核心入口，注意要让用户觉得在爽玩，没有创作负担
- **聊天为主导的 REMIX 功能** —— 主要是做立绘二创，聊天倒不重要

---

### 2. 技术侧 —— 真正打通三个端

- 每个端都彻底 Agent Native 化，也就是主控模型可以同时掌握三端的过程、结果和体验
- 比如 REMIX，应该是游戏侧提需求，剧本侧生产给视频侧，最后游戏侧 mock 后上线
- 现在的管线里，REMIX 的剧本还是放在游戏侧的，这样会导致游戏侧臃肿，并且转来转去的。REMIX 可以不动现在的剧本侧的管线，但是应该是一套单独的服务，独立化，并且自己 mock 游戏侧的体验。游戏侧只拿数据和校验。
- 其他的就是对上前面产品侧的需求进行快速的改造
- 工具的 GUI 只要能用就行

---

### 3. 运营侧 —— 开始运营真实用户，拒绝闭门造车

- 这个点很重要，我们动作太慢了，4 个月过去了，demo 都没有，现在需要尽快上 demo，尽快圈住用户
- 开源计划等核心 PR 甚至都没排上时间去做

---

## 接下来的几个动作

1. 现有雨佳和何莲的工作继续，保障视频生产，手搓就手搓了，可以合适范围内继续拔高下剧本质量
2. 运营和 UIUX 先行，反正也要上一个版本，先测起来，后面要做的功能，UIUX 都得先行起来，不能到最后再做 UIUX
3. 不需要 PRD 稿，直接在 demo 上做 feature，现在 demo 上的 feature 已经领先很多，就算要删要改问题也不大。代码不值钱，不要吝啬做 feature。
4. 技术侧，需要进行灵活的调整，将基建做足
5. 新的剧本项目快速测起来，这套对后续技术搭建有很大作用
6. 所有代码 Atomic commits，一次只做一件事，后面就算改出问题也随时可以让 AI 回退，接受让产品侧和运营侧也参与提交 commits 进行简单的 debug 和修改，而不是卡在反映问题修 bug，我们大部分时间不是在做一个上线的商业产品，是内部工具和 demo
